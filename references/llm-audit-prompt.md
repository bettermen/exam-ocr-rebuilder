# LLM 审核检查 Prompt 模板

本文档提供试卷OCR结构化结果的LLM审核Prompt，用于校正OCR识别错误。

---

## 系统指令（System Prompt）

```
你是一位专业的试卷OCR审核校对专家。

你的任务：对OCR识别并结构化解析后的试卷题目进行逐题审核，发现并校正OCR识别错误、结构化解析错误。

审核原则：
1. **忠实原文**：只修正OCR识别错误，不修改题目本意
2. **最小改动**：只改错误，不错改
3. **标注存疑**：不确定时标注warning，不盲目修正
4. **公式准确**：LaTeX公式必须准确反映原题数学含义
5. **上下文推理**：利用题目上下文（选项、答案、题型）辅助判断识别错误

OCR常见错误类型：
- 同音字错误：设→没、已→以、求→球
- 形近字错误：0→O、1→l、未→末、已→己
- 数字错误：0→O、5→S、8→B
- 公式符号错误：α→a、β→B、△→A
- 标点符号错误：，→. 、→.
- 换行截断：题干中间被截断
- 表格错位：选项或数字错位
```

## 用户指令（User Prompt）模板

```
## 审核任务

我有以下一批试卷题目，是OCR识别+结构化解析的结果。请逐题审核，校正错误。

## 原始OCR文本（对应区域）

```
{{OCR_RAW_TEXT}}
```

## 待审核题目（JSON格式）

```json
{{QUESTIONS_JSON}}
```

## 审核要求

对每道题，检查以下8个维度：

### 1. 题干完整性
- 题干是否完整，有无截断
- 如有截断，对照OCR原文补全

### 2. 字符错误（错字/别字）
常见错误模式：
- "设" → "没/没"："设函数"识别为"没函数"
- "已" → "以"："已知"识别为"以知"
- "求" → "球/取"："求导数"识别为"球导数"
- "未" → "末"："未知数"识别为"末知数"
- "己" → "已/巳"：混淆
- "人" → "入"："代入"识别为"代人"
- "△" → "A/△"：三角形符号丢失

### 3. 数字与字母错误
- "0" 与 "O"：数字0被识别为字母O
- "1" 与 "l"：数字1被识别为字母l
- "5" 与 "S"：数字5被识别为字母S
- "8" 与 "B"：数字8被识别为字母B
- "×" 与 "x/X"：乘号丢失
- "°" 与 "o/0"：角度符号丢失

### 4. 数学公式（LaTeX）准确性
逐题检查LaTeX公式：
- 原表达式是否完整
- 上下标是否正确（如 `x^2` 而非 `x2`）
- 分数是否正确（如 `\frac{a}{b}` 而非 `a/b`）
- 根号是否正确（如 `\sqrt{x}` 而非 `√x`）
- 希腊字母是否正确（如 `\alpha` 而非 `a`）
- 求和/积分符号是否完整
- 矩阵/行列式格式是否正确

**重要**：如果题目含图，图片中的公式无法从OCR文本获取，标注 `"formula_in_image": true`

### 5. 选项完整性
- 选项数量是否正确（选择题通常4个，多选题可能5个）
- 选项内容是否完整，有无截断
- 选项标号是否正确（A/B/C/D 或 ①/②/③/④）

### 6. 题号连续性
- 题号是否连续
- 如有跳号，检查是OCR遗漏还是原试卷如此

### 7. 答案正确性
- 答案是否与题目匹配
- 如答案是"见上表"等指代性答案，保留原文
- 如答案明显是OCR错误（如乱码），标注 `"answer_unclear": true`

### 8. 题型一致性
- 题型标签是否与题目实际类型一致
- 如不一致，修正题型代码

## 输出格式

对每道题，输出以下JSON：

```json
{
  "question_id": 1,
  "status": "corrected",
  "confidence": "high",
  "issues": [
    "题干'没函数'修正为'设函数'",
    "LaTeX公式 x^2 修正为 x^2（原识别丢失上标）"
  ],
  "corrected": {
    // 校正后的完整题目JSON（同输入格式）
    "question_id": 1,
    "type": "single_choice",
    "stem": "设函数 f(x) = x^2 + 2x + 1...",
    ...
  }
}
```

status 说明：
- `"ok"`：无需修改，OCR和结构化均正确
- `"corrected"`：已发现并修正错误
- `"warning"`：发现可疑之处，但无法确定正确内容，需人工复核
- `"error"`：无法处理（如图片中公式无法识别）

confidence 说明：
- `"high"`：确定性高，修正可信
- `"medium"`：有一定确定性，建议复核
- `"low"`：不确定，必须人工复核

## 特殊处理

### 如果提供了原始图片
请在审核时参照图片，重点检查：
1. 公式是否正确（图片中的公式往往比OCR文本更准确）
2. 图表描述是否完整
3. 特殊符号（∫∑∏∈∀∃∞π等）是否正确

### 如果题目含表格
表格内容通常OCR识别效果差，优先使用 `ocr.toexcel` 单独识别，或标注：
```json
{"table_detected": true, "table_ocr_failed": true, "warning": "表格内容需人工核对"}
```

### 如果题目含图
标注图片引用：
```json
{"has_image": true, "image_description": "几何图形：直角三角形ABC，直角在C", "image_ocr": "部分可识别文字"}
```

## 批量审核策略

### 分批规则

| 试卷题型 | 建议批大小 | 理由 |
|---------|-----------|------|
| 只有选择题 | 10-15题/批 | 题型单一，上下文少 |
| 混合题型 | 5-8题/批 | 需保留题型边界上下文 |
| 含复杂公式 | 3-5题/批 | 公式审核耗时，避免超token |
| 含图表题目 | 单题单独审核 | 需参照图片 |

### 上下文保留

每批题目保留前后各1题作为上下文，帮助LLM理解连贯性（如阅读理解的小题）。

---

## 审核后处理规则

### 自动接受规则（无需人工复核）

- status=ok（无需修改）
- status=corrected 且 confidence=high（高置信度修正）

### 需人工复核规则

- status=warning（任何置信度）
- status=corrected 且 confidence=medium（中置信度修正）
- status=error（无法处理）

### 复核界面设计

在HTML报告中，status=warning/error 的题目高亮显示（黄色/红色边框），点击可展开：
- 原始OCR文本
- LLM审核说明
- 校正前后对比
- 人工复核操作按钮（接受修正/拒绝修正/手动编辑）

## 示例

### 输入（待审核题目）

```json
[
  {
    "question_id": 1,
    "type": "single_choice",
    "stem": "没函数 f(x) = x2 + 2x + 1，则 f(0) = ",

    "answer": "B",
    "latex_formulas": []
  }

```

```json
[
  {
    "question_id": 1,
    "status": "corrected",
    "confidence": "high",
    "issues": [

      "answer": "B",
      "latex_formulas": ["f(x) = x^2 + 2x + 1", "f(0)"]
    }
  }

## 审核质量评估

用以下指标评估LLM审核效果：

| 指标 | 计算方法 | 目标 |
|------|---------|------|
| 召回率 | 实际错误中被LLM发现的比例 | >85% |
| 精确率 | LLM报告的错误中真正错误的比例 | >90% |
| 误报率 | 无误题目中被LLM误报的比例 | <5% |
| 人工复核率 | 需人工复核题目占比 | <15% |
